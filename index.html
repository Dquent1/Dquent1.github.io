Définition : Une page web est codé avec le langage HTML qui structure les éléments mais aussi est codé grâce au CSS qui permet de réaliser la mise en page. Le HTML est un langage balisé, chaque élément est défini par une balise ouvrante < balise > ainsi qu'une balise fermante < /balise > Les balises peuvent être imbriquées les unes dans les autres tant que la structure de base est respectée

Structure d'une page web.

La structure de base d'une page web est constitué de plusieurs éléments balisés essentiels à l'interpretation du code source et doit contenir

  • La description STRICTEMENT nécessaire du type du document définie par : < !DOCTYPE html >
  • une balise < html > ....< /html > qui est la balise définissant les limite du code HTML et se doit de contenir la balise suivant dans la liste
  • une balise < head > .... < /head > qui permet de défninir le nom de référence de la page web et peut contenir d'autre informations de type "meta" qui vont agir sur le document
  • la balise < head > contient de ce fait la balise title qui définie le nom de référence de la page. ici le nom de la page est <title> Cours initiation HTML et CSS </title> Attention une page web a un seul et unique "title"
  • Pour finir une page html doit contenir une balise < body > .... < /body > qui définir le corps/contenu de la page web

Exemple de structure de base:

				< !DOCTYPE html >
				< html > 
					< head > 
						<title> Cours initiation HTML et CSS  </title>
					< /head >
					< body >
						Contenu
					< /body >
				< /html >

Les balises du sémantiques < body > .... < /body >

  • < h1 > ... < h1 > qui définit les "headers" ou intitulés h1, h2, h3, h4, h5, h6. Le header h1 a la police la plus grande police parmis es header. Par convention il n'en faut pas plus de 6 par pages
  • < p > ... < /p > qui définit un bloc text avec un saut de ligne à la fin du bloc.
  • < em > ... < /em > qui rend italique le texte contenu dans cette balise et permet de conserver l'aspect sémantique du texte contenu dans cette balise.
  • < strong > ... < /strong > qui rend gras le texte contenu dans cette balise et permet de conserver l'aspect sémantique du texte contenu dans cette balise.
  • < mark > ... < /mark >qui rend surligne le texte contenu dans cette balise et permet de conserver l'aspect sémantique du texte contenu dans cette balise.

Les balises de liste < ol > et < ul >

  • les balises < ol >< /ol > définissent un listing ordonnée et chaque élément de cette liste doit être balisé par < li > 1- ... < /li > par exemple sur cette page web, chaque parties du cours sont contenues dans une liste ordonnée et chaque partie est déterminé par un numéro
  • les balises < ol >< /ol > définissent un listing desordonnée et chaque élément de cette liste doit être balisé par < li > ... < /li > par exemple sur cette page web, chaque sous parties du cours sont contenues dans une liste desordonnée et chaque partie est déterminé par une puce

Les balises < pre > et < code >

Ces balises permettent d'unifier la police globale du texte contenu dans celles-ci. Cette méthode s'écrit de la manière suivante sous forme de code :

< pre > < code > votre bloc à unifier < /code > < /pre >

Les tableaux

  • Les tableaux en HTML sont déclarée par les balises <table> </table>
  • L'entête du tableaux désigne la première ligne par les balises <thead> </thead>
  • Le pied du tableaux désigne la dernière ligne par les balises <thead> </thead>
  • les colonnes sont en réalité définies par le nombre de cases successives que vous déclarer. Dans le cas des lignes de pied et d'entête, ces case son déclaré de la manière suivante <th> </th>
  • Le lignes du intermédiaires du tableaux désignant les lignes contenues entre les lignes d'entête et les lignes de pied du tableau. ces lignes sont déclarés par <tr> </tr>
  • Dans le cas des lignes intermédiare du tableau, les colonnes sont définies par le nombre de cases successives que vous déclarer <td> </td>
case 1 thead case 2 thead
case1 tfoot case2 tfoot
case 1 inter case 3 inter

Ce tableau ci-dessus, est codé selon les règles énoncées précedement de la façon suivante.

					<table>
					<thead>
						<th> case 1 thead </th>
						<th> case 2 thead </th>
						<th></th>
					</thead>
					<tfoot >
						<th> case1 tfoot </th>
						<th> case2 tfoot </th>
						<th></th>
					</tfoot>
						<tr>
							<td>case 1 inter</td>
							<td>case 2 inter</td>

Attention en absence de thead et/ou tfoot le tableau et défini par défaut par les balise <tr> </tr> et <td> </td>

Aspect de mise en forme et CSS

Dans cette page d'initiation, aucune balises de mise en forme a été utilisée car elles biaisent le caractère sémantique du contenu de ces balises. Ces balises sont devenues actuellement obsolètes et c'est le langage CSS qui permet de définir la mise en page. la mise en page sans CSS est en réalité la mise en page CSS par défaut. Cela signifie que le texte contenu dans le corps de la page web va prendre tous l'espace disponible pour être afficher. Afin d'illustrer ce principe, je vous laisse jouer avec la taille de votre fenêtre web vous constaterez que le texte s'adapte à la taille de votre fenêtre. Seul l'implémentation de code CSS permet de réaliser une mise en fome particulière et pouvant être statique
