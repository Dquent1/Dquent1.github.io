Initiation au language HTML et CSS

définition : Une page web est codé avec le langage HTML qui structure les éléments mais aussi est codé grâce au CSS qui permet de réaliser la mise en page. Le HTML est un langage balisé, chaque élément est défini par une balise ouvrante < balise > ainsi qu'une balise fermante < /balise > Les balises peuvent être imbriquées les unes dans les autres tant que la structure de base est respectée

  1. Structure d'une page web.

    2. La structure de base d'une page web est constitué de plusieurs éléments balisés essentiels à l'interpretation du code source et doit contenir

    • La description STRICTEMENT nécessaire du type du document définie par : < DOCTYPE html >
    • une balise < html > ....< /html > qui est la balise définissant les limite du code HTML et se doit de contenir la balise suivant dans la liste
    • une balise < head > .... < /html > qui permet de défninir le nom de référence de la page web et peut contenir d'autre informations de type "meta" qui vont agir sur le document
    • la balise < head > contient de ce fait la balise title qui définie le nom de référence de la page. ici le nom de la page est <title> Cours initiation HTML et CSS </title> Attention une page web a un seul et unique "title"
    • Pour finir une page html doit contenir une balise < body > .... < /body > qui définir le corps/contenu de la page web

  2. importation d'encodage.

    • Les caratère spéciaux comme es lettre accentué nécéssite "l'importation d'un type unicode. Cette action se réalise dans la partie < head > .... < /html > <meta http-equiv="Content-Type" content="text/html; charset=UTF-8" />

  3. Les balises du < body > .... < /body >

    • < h1 > ... < h1 > qui définit les "headers" h1, h2, h3, h4, h5, h6. Le header h1 a la police la plus grande police parmis es header. Par convention il n'en faut pas plus de 6 par pages
    • < p > ... < /p > qui définit un bloc text avec un saut de ligne à la fin du bloc.
    • < em > ... < /em > qui rend italique le texte contenu dans cette balise et permet de conserver l'aspect sémantique du texte contenu dans cette balise.
    • < strong > ... < /strong > qui rend gras le texte contenu dans cette balise et permet de conserver l'aspect sémantique du texte contenu dans cette balise.
    • < mark > ... < /mark >qui rend surligne le texte contenu dans cette balise et permet de conserver l'aspect sémantique du texte contenu dans cette balise.
    • < > ... < >

  4. Les balises de liste < ol > et < ul >

    • les balises < ol >< /ol > définissent un listing ordonnée et chaque élément de cette liste doit être balisé par < li > 1- ... < /li > par exemple sur cette page web, chaque parties du cours sont contenues dans une liste ordonnée et chaque partie est déterminé par un numéro
    • les balises < ol >< /ol > définissent un listing desordonnée et chaque élément de cette liste doit être balisé par < li > ... < /li > par exemple sur cette page web, chaque sous parties du cours sont contenues dans une liste desordonnée et chaque partie est déterminé par une puce

  5. Les définitions < dt > et < df >

    • les définitions de termes < dt >< /dt > permet d'énumerer des termes à définir par les balises de définition décrites ci-dessous

    • les définitions de termes < dt >< /dt > permet de définir les termes à définir contenues dans les balises < dt >< /dt >

  6. Les BALISES < pre > et < code >

    7. Ces balisent permette d'unifier la police globale du texte contenu dans celles-ci. Cette méthode s'écrit de la manière suivante sous forme de code :

    < pre > < code > votre bloc à unifier < /code > < /pre >

  7. Aspect de mise en forme et CSS

    8. Dans cette page d'initiation, aucune balises de mise en forme a été utilisée car elles biaisent le caractère sémantique du contenu de ces balises. Ces balises sont devenues actuellement obsolètes et c'est le langage CSS qui permet de définir la mise en page. la mise en page sans CSS est en réalité la mise en page CSS par défaut. Cela signifie que le texte contenu dans le corps de la page web va prendre tous l'espace disponible pour être afficher. Afin d'illustrer ce principe, je vous laisse jouer avec la taille de votre fenêtre web vous constaterez que le texte s'adapte à la taille de votre fenêtre. Seul l'implémentation de code CSS permet de réaliser une mise en fome particulière et pouvant être statique
